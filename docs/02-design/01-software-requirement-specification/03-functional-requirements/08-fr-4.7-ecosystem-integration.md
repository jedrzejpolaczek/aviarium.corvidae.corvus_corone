# FR §4.7 — Ecosystem Integration

*Enables export of completed Experiment data to external benchmark platforms (COCO, IOHprofiler, Nevergrad) with explicit information-loss documentation.*
*Source: MANIFESTO Principle 24, 26.*
*Use Cases: UC-06.*

---

## FR-23

**The system MUST support export of completed Experiment data in at least one external benchmark platform format (COCO, IOHprofiler `.dat`, Nevergrad). The format mapping used for each target MUST be applied exactly as documented in `docs/03-technical-contracts/01-data-format.md` §4.**

- Source: MANIFESTO Principle 26
- Connects to: `docs/02-design/01-software-requirement-specification/02-use-cases/07-uc-06.md`; `docs/03-technical-contracts/01-data-format.md` §4; `docs/03-technical-contracts/02-interface-contracts.md` Ecosystem Bridge interface

## FR-24

**Before producing any export file, the system MUST generate an information-loss manifest that lists every field in the Corvus Corone schema that has no equivalent in the target format, or that is approximated or dropped in the conversion. The manifest MUST be delivered to the Researcher before the export is confirmed.**

- Source: MANIFESTO Principle 26
- Connects to: `docs/02-design/01-software-requirement-specification/02-use-cases/07-uc-06.md` Steps 3–4; `docs/03-technical-contracts/01-data-format.md` §4

## FR-25

**The system MUST reject export requests for formats not listed in its set of supported mappings. If a mandatory target field is not present in the Run data, the system MUST reject the export and identify the missing field explicitly. The system MUST NOT produce a partial export without an information-loss manifest.**

- Source: MANIFESTO Principle 26
- Connects to: `docs/02-design/01-software-requirement-specification/02-use-cases/07-uc-06.md` F1, F2; `docs/02-design/01-software-requirement-specification/03-functional-requirements/03-fr-4.2-algorithm-registry.md` FR-05 (registration prerequisite)

## FR-26

**All format mappings used for ecosystem export MUST be documented in `docs/03-technical-contracts/01-data-format.md` §4 and MUST be versioned alongside the data schema. Undocumented mappings MUST NOT be applied.**

- Source: MANIFESTO Principle 24
- Connects to: `docs/03-technical-contracts/01-data-format.md` §4; `docs/02-design/01-software-requirement-specification/02-use-cases/07-uc-06.md`; `docs/05-community/02-versioning-governance.md`
