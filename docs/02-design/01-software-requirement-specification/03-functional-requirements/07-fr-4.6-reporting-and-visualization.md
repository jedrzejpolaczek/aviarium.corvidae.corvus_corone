# FR §4.6 — Reporting & Visualization

*Generates Researcher Reports, Practitioner Reports, and Raw Data exports from completed Experiments.*
*Source: MANIFESTO Principles 24–25; Constraints CONST-SCI-01, CONST-SCI-05, CONST-SCI-06, CONST-COM-03.*
*Use Cases: UC-01 Steps 9–10; UC-03.*

---

## FR-20

**The system MUST generate a Researcher Report and a Practitioner Report for every completed Experiment. Each report MUST scope all conclusions explicitly to the Problem Instances and Algorithm Instances tested in that Experiment.**

- **Priority:** MUST
- **Source:** MANIFESTO Principle 25
- **Accepted when:** A completed Experiment produces two distinct report objects, one of type `ResearcherReport` and one of type `PractitionerReport`; each report contains a `scope` field explicitly naming the Algorithm Instance IDs and Problem Instance IDs tested.
- Connects to: `docs/02-design/01-software-requirement-specification/02-use-cases/02-uc-01.md` Step 9; `docs/02-design/01-software-requirement-specification/02-use-cases/04-uc-03.md`; `docs/04-scientific-practice/01-methodology/01-benchmarking-protocol.md` Steps 7–8

## FR-21

**Every system-generated report MUST include a mandatory limitations section that: (a) states the scope conditions of every conclusion, (b) identifies characteristics and problem classes for which no data was collected, and (c) does NOT produce or display global algorithm rankings or "best algorithm" declarations.**

- **Priority:** MUST
- **Source:** MANIFESTO Principle 24
- **Accepted when:** (a) Every generated report contains a non-empty `limitations` section; (b) a report generation path that omits the limitations section is rejected at the template level; (c) no report template produces a ranking table or "best algorithm" string.
- Connects to: `docs/02-design/01-software-requirement-specification/05-constraints/01-constraints.md` CONST-SCI-01, CONST-SCI-05, CONST-SCI-06; `docs/02-design/01-software-requirement-specification/02-use-cases/04-uc-03.md` Step 4

## FR-22

**The system MUST produce a Raw Data export alongside every completed Experiment's reports. The export format MUST be open, machine-readable, and readable without proprietary tools, in compliance with CONST-COM-03.**

- **Priority:** MUST
- **Source:** MANIFESTO Principles 20, 25
- **Accepted when:** The Raw Data export produced for a completed Experiment is valid JSON or CSV parseable by the Python standard library (`json` or `csv` module) without additional dependencies; the export is present in the same output directory as the reports.
- Connects to: `docs/02-design/01-software-requirement-specification/05-constraints/01-constraints.md` CONST-COM-03; `docs/02-design/01-software-requirement-specification/04-non-functional-requirements/01-non-functional-requirements.md` NFR-OPEN-01; `docs/03-technical-contracts/01-data-format.md` §3
