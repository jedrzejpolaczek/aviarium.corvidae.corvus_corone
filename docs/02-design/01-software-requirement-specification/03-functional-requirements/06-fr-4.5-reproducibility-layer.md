# FR §4.5 — Reproducibility Layer

*Versions all Artifacts, supports open data publication, and enables long-term re-execution.*
*Source: MANIFESTO Principles 19–22; ADR-001.*
*Use Cases: UC-05.*

---

## FR-17

**Every entity (Study, Experiment, Run, Algorithm Instance, Problem Instance, Result Aggregate) MUST carry a globally unique identifier in UUID format. Local file paths MUST NOT serve as entity identifiers.**

- **Priority:** MUST
- **Source:** MANIFESTO Principles 19, 21; ADR-001 (server-compatibility constraint)
- **Accepted when:** Every stored entity's `id` field parses as a valid RFC 4122 UUID; no entity record contains a field whose value is a file path or directory name used as an entity reference.
- Connects to: `docs/03-technical-contracts/01-data-format/01-index.md`; `docs/02-design/02-architecture/01-adr/adr-001-library-with-server-ready-data-layer.md`

## FR-18

**The system MUST be capable of producing an Artifact archive for any completed Experiment containing: Study plan record, all Algorithm Instance records (with `code_reference`), all Problem Instance records (versioned), all Run records (with seeds and `execution_environment`), and all Result Aggregates.**

- **Priority:** MUST
- **Source:** MANIFESTO Principle 19
- **Accepted when:** `experiment.archive()` produces a self-contained archive whose contents include one `StudyRecord`, at least one `AlgorithmInstance` (with a non-null `code_reference`), at least one `ProblemInstance` (with a `version` field), all `RunRecord` entries (each with `seed` and `execution_environment`), and all `ResultAggregate` entries.
- Connects to: `docs/03-technical-contracts/01-data-format/04-study.md` §2.3 – `docs/03-technical-contracts/01-data-format/08-result-aggregate.md` §2.7; `docs/04-scientific-practice/01-methodology/01-benchmarking-protocol.md` Step 8

## FR-19

**Cross-entity references MUST use entity IDs only — no entity field may reference another entity by file path, directory structure, or local naming convention.**

- **Priority:** MUST
- **Source:** ADR-001 (server-compatibility constraint)
- **Accepted when:** A schema validator scanning all cross-entity reference fields (`study_id`, `experiment_id`, `problem_id`, `algorithm_id`, `run_id`) finds no value that contains `/`, `\`, `.json`, `.yaml`, or any other path component.
- Connects to: `docs/03-technical-contracts/01-data-format/01-index.md`; `docs/03-technical-contracts/02-interface-contracts/01-index.md`
