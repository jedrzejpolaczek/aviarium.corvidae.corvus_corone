## FR-01

**The system MUST store Problem Instance records conforming to the schema defined in `docs/03-technical-contracts/01-data-format.md` ┬¦2.1, including all required fields: `id` (UUID), `name`, `version`, search space descriptors, objective specification, evaluation budget, landscape characteristics, and provenance.**

- **Priority:** MUST
- **Source:** MANIFESTO Principles 4, 7
- **Accepted when:** A `ProblemInstance` with all required fields is stored and retrieved without data loss; a registration attempt missing any required field is rejected with a `ValidationError` that names the missing field.
- Connects to: `docs/03-technical-contracts/01-data-format.md` ┬¦2.1; `docs/03-technical-contracts/02-interface-contracts.md` Problem interface

## FR-02

**The system MUST validate Problem Instance completeness on registration: `dimensions` must equal `len(variables)`; variable bounds must be valid (lower < upper for continuous); all required fields must be present. Invalid submissions MUST be rejected with a specific error message identifying the violation.**

- **Priority:** MUST
- **Source:** MANIFESTO Principle 7
- **Accepted when:** Each of the following raises a `ValidationError` whose message identifies the specific violation: (a) `dimensions=2` with `len(variables)=3`; (b) a `ContinuousVariable` with `lower=1.0, upper=0.5`; (c) a record with `name=None`.
- Connects to: `docs/03-technical-contracts/01-data-format.md` ┬¦2.1 validation rules

## FR-03

**The system MUST support Problem Instance versioning ŌĆö modifying a Problem Instance produces a new version rather than overwriting the existing record. A Study references a specific version; that version MUST remain retrievable for the lifetime of the Study.**

- **Priority:** MUST
- **Source:** MANIFESTO Principles 6, 21
- **Accepted when:** Modifying a stored `ProblemInstance` produces a new record with a distinct `id` and incremented `version`; the original record remains retrievable by its original `id`; a Study referencing `version="1.0.0"` cannot use `version="2.0.0"` of the same problem.
- Connects to: `docs/03-technical-contracts/01-data-format.md` ┬¦2.1; `docs/05-community/02-versioning-governance.md`

## FR-04

**The system MUST prevent inclusion of a Problem Instance version that differs from the version declared in the parent Study plan.**

- **Priority:** MUST
- **Source:** MANIFESTO Principle 21
- **Accepted when:** Attempting to execute a Study that declares Problem Instance version `v1.0.0` when the registry contains only `v2.0.0` raises a `VersionMismatchError` before any Run begins.
- Connects to: `docs/03-technical-contracts/01-data-format.md` ┬¦2.3 (Study), ┬¦2.1 (Problem Instance)
