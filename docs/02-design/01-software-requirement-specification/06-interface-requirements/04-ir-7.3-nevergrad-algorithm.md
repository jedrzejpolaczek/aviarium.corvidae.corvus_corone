# §7.3 — Nevergrad Algorithm Interface

**Direction:** Import (Nevergrad optimizer → Corvus Corone Algorithm Instance) — bidirectional during a Run
**Format:** In-process Python adapter; no serialization format (ask/tell API calls)
**Protocol:** In-process Python function calls; `ask()` → `tell(candidate, loss)` per evaluation
**Requirement:** FR-25
**Data mapping:** [docs/03-technical-contracts/01-data-format/11-interoperability-mappings.md §4.3](../../../03-technical-contracts/01-data-format/11-interoperability-mappings.md)
**Implementation:** `corvus_corone.algorithms.adapters.NevergradAdapter`
**Dependency:** `nevergrad` optional package (`pip install nevergrad`); not a hard dependency of the core
**Status:** ✅ Mapping complete (REF-TASK-0006)

---

## Method Mapping

| Corvus Algorithm Interface | Nevergrad call | Notes |
|---|---|---|
| `initialize(search_space, seed)` | `ng.optimizers.registry[name](parametrization, budget)` | Fresh optimizer per Run; seed via `parametrization.random_state.seed(seed)` |
| `suggest(context, batch_size)` | `optimizer.ask()` × batch_size | Candidates queued in FIFO deque for pairing with `observe()` |
| `observe(solution, result)` | `optimizer.tell(candidate, loss)` | FIFO dequeued; loss negated for maximization |
| `get_supported_variable_types()` | — | Returns `["continuous", "integer", "categorical"]` |
| `get_metadata()` | `nevergrad.__version__` | Returns full Algorithm Instance record |

## SearchSpace Variable Mapping

| Corvus variable type | Nevergrad parametrization | Notes |
|---|---|---|
| `continuous` | `ng.p.Scalar(lower=lo, upper=hi)` | Bounds inclusive |
| `integer` | `ng.p.Scalar(lower=lo, upper=hi).set_integer_casting()` | Bounds inclusive |
| `categorical` | `ng.p.Choice(choices)` | Unordered discrete set |

Full field-level mapping, boilerplate count, and manifest keys (LOSS-NG-01 through LOSS-NG-07) are in
[§4.3 of the interoperability mappings](../../../03-technical-contracts/01-data-format/11-interoperability-mappings.md).

## Behavior

- A Nevergrad optimizer is registered as an Algorithm Instance by constructing a `NevergradAdapter` and passing it to the Algorithm Registry
- The adapter translates Corvus `SearchSpace.variables` to an `ng.p.Dict` parametrization in `initialize()`
- The `initialize()` call creates a fresh optimizer for every Run, satisfying the §2 isolation requirement (no state leaks between Runs)
- The wrapped optimizer participates in Studies identically to a native Algorithm Instance
- The Algorithm Instance record's `framework` field is set to `"nevergrad"` and `framework_version` is pinned at registration time
- For maximization problems, `observe()` negates the loss before `tell()` because Nevergrad always minimizes internally (LOSS-NG-04)
- `nevergrad` is imported lazily in `initialize()`; importing `NevergradAdapter` does not fail when `nevergrad` is not installed

## Failure Handling

- If `nevergrad` is not installed, `initialize()` raises `ImportError` with an actionable install message; the Study does not start
- If the Nevergrad optimizer raises any exception during `suggest()` or `observe()`, it propagates as a Run failure (FR-12); the Study continues with remaining Runs
- If `suggest()` or `observe()` is called before `initialize()`, a `RuntimeError` is raised immediately
- Unsupported variable types (not `continuous`, `integer`, or `categorical`) raise `ValueError` in `initialize()` with the unsupported type name
