# UC-11: Learner Explores Researcher Study Results

**Actor:** Learner
**Trigger:** Wants to explore completed Researcher study results as educational material to understand algorithm behaviour in practice
**Goal:** Consume benchmarking results with interpretive support — understand what the results mean, what they do not mean, and what questions they open up — without the system drawing conclusions on the Learner's behalf

---

## Diagram

```mermaid
sequenceDiagram
    actor Learner
    participant Results as Study Results
    participant Interp as Interpretation Support
    participant Socratic as Socratic Guide

    Learner->>Results: Browse available completed Studies
    Results-->>Learner: Study list (title, algorithms compared, problem characteristics, date)

    Learner->>Results: Select Study
    Results-->>Learner: Practitioner Report (scoped summaries) + Researcher Report (statistical detail)

    Learner->>Interp: "What does this result mean?"
    Interp->>Results: Retrieve result context (scope statement, limitations section, tested problem set)
    Interp-->>Learner: Plain-language interpretation grounded in the result's explicit scope

    Learner->>Interp: "Does this mean Algorithm A is always better?"
    Interp-->>Learner: Scope correction — result applies only to tested Problem Instances; references limitations section (Principle 3)

    alt Learner wants to reason further
        Learner->>Socratic: "Why might Algorithm A perform better here?"
        Socratic-->>Learner: Guided question (UC-09)
    else Learner wants visualisation
        Learner->>Results: Request convergence or comparison visualisation
        Results-->>Learner: VIZ-L1-01..04 plots from completed Study (UC-07)
    end

    Learner->>Interp: "What questions does this result leave open?"
    Interp-->>Learner: Unresolved questions implied by the scope and limitations (Principle 3)
```

---

## Preconditions

- At least one completed Study with a generated Report exists in the system
- The Learner has read-only access to Reports and ResultAggregates (no modification rights)

## Main Flow

1. Learner browses the list of completed Studies; the system displays each Study's title, the algorithms compared, the problem characteristics of the test set, and the study date — enough to decide which Study is relevant to their learning goal
2. Learner selects a Study and receives both the Practitioner Report (scoped summaries) and the Researcher Report (full statistical detail with explicit limitations section)
3. Learner asks what a result means in plain language ("What does a higher ANYTIME-ECDF_AREA mean?"); the system answers by grounding the explanation in the result's scope statement and the metric's definition in the metric taxonomy
4. If the Learner attempts to generalise beyond the result's scope ("Algorithm A is always better"), the system issues a scope correction referencing the limitations section and the specific Problem Instances tested — it does not confirm the over-generalisation (MANIFESTO Principle 3)
5. System highlights the limitations section of the Report as the authoritative boundary on what the result can and cannot conclude
6. Learner can pivot to:
   - **Socratic mode** (UC-09) to reason about why the result came out the way it did
   - **Algorithm visualisation** (UC-07) to see the algorithm's behaviour on the tested problems
   - **Contextual help** (UC-08) to understand the algorithms compared in the Study
7. Learner can ask what questions this result leaves open; the system identifies the unresolved questions implied by the Study's scope and limitations (problem characteristics not covered, budgets not tested, noise conditions absent from the test set)

## Postconditions

- Learner has read and interpreted at least one Study result with interpretive support
- No conclusion has been stated by the system that goes beyond the Report's explicit scope
- The Learner has been shown the path to deeper understanding via UC-07 (visualisation), UC-08 (contextual help), or UC-09 (Socratic reasoning)

## Failure Scenarios

- *F1: No completed Studies available* — System notes that no completed Studies are available and redirects the Learner to UC-07 (algorithm visualisation from metadata only) or UC-08 (contextual help without empirical examples)
- *F2: Learner attempts to modify a Report* — System returns a permissions error; Reports are immutable after generation (MANIFESTO Principle 21); Learner can annotate privately but the canonical Report is read-only
- *F3: Learner requests a conclusion the result cannot support* — System provides a scope-corrected explanation; it does not state a conclusion that over-reaches the tested Problem Instance set (Principle 3)
