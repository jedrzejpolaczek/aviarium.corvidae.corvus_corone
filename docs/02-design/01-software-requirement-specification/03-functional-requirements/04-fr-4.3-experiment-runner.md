## FR-08

**The system MUST enforce the pre-registration gate: the Study's problem set, algorithm set, experimental design, and hypotheses MUST be locked before any Run is executed. Any attempt to modify these fields after locking MUST be rejected and logged with a timestamp.**

- **Priority:** MUST
- **Source:** MANIFESTO Principle 16; `docs/04-scientific-practice/01-methodology/01-benchmarking-protocol.md` Steps 1ŌĆō5
- **Accepted when:** Modifying any of `problem_ids`, `algorithm_ids`, `repetitions`, `budget`, or `pre_registered_hypotheses` on a locked `StudyRecord` raises `StudyLockedError`; the rejection is written to the system log with an ISO 8601 timestamp.
- Connects to: `docs/03-technical-contracts/01-data-format.md` ┬¦2.3 (Study)

## FR-09

**The system MUST assign all Run seeds ŌĆö the Researcher MUST NOT control individual seed values. Seeds MUST be assigned deterministically from the Study's declared `seed_strategy` before any Run begins.**

- **Priority:** MUST
- **Source:** MANIFESTO Principle 18
- **Accepted when:** The same Study executed twice with the same `seed_strategy` assigns identical seeds to identical `(problem_id, algorithm_id, repetition_index)` triplets; the public API exposes no parameter for setting individual seed values.
- Connects to: `docs/03-technical-contracts/01-data-format.md` ┬¦2.5 (Run); `docs/04-scientific-practice/01-methodology/01-benchmarking-protocol.md` Step 6

## FR-10

**The system MUST record the full execution environment for each Experiment: operating system, hardware specification, Python version, and dependency versions. This record MUST be captured automatically, not entered manually.**

- **Priority:** MUST
- **Source:** MANIFESTO Principle 19
- **Accepted when:** An `ExperimentRecord` produced without specifying environment parameters contains non-null `execution_environment` fields for `os`, `python_version`, and `dependencies` (captured from the live runtime).
- Connects to: `docs/03-technical-contracts/01-data-format.md` ┬¦2.4 (Experiment.execution_environment)

## FR-11

**Each Run MUST execute without any shared mutable state with other Runs in the same Experiment. The runner MUST enforce process isolation or equivalent isolation guarantee.**

- **Priority:** MUST
- **Source:** MANIFESTO Principle 18
- **Accepted when:** An algorithm implementation that stores state in a module-level variable produces identical results across all repetitions when executed by the Runner, confirming the Runner resets or isolates state between Runs.
- Connects to: `docs/03-technical-contracts/01-data-format.md` ┬¦2.5 (Run); `docs/03-technical-contracts/02-interface-contracts.md` Runner interface

## FR-12

**The system MUST record a failed Run with its failure reason and timestamp rather than silently skipping it. A Study with failed Runs MUST NOT be reported as if all planned Runs completed.**

- **Priority:** MUST
- **Source:** MANIFESTO Principle 19
- **Accepted when:** A Run whose `evaluate()` raises an uncaught exception produces a `RunRecord` with `status="failed"` and a non-empty `failure_reason`; the resulting `ExperimentRecord` reflects the count of failed vs. completed Runs; `generate_report()` on an Experiment with failed Runs includes a warning rather than silently presenting only the succeeded Runs.
- Connects to: `docs/03-technical-contracts/01-data-format.md` ┬¦2.5 (Run.status, Run.failure_reason)
